## 🎨 视觉规范（赛博朋克系）

### 配色系统

```css
/* 主色板 */
--bg-primary: #0a0a0a;        /* 深色模式，纯黑背景 */
--accent-cyan: #00f5ff;       /* 霓虹青，主强调色 */
--accent-magenta: #ff0080;    /* 品红，对比色 */
--text-primary: #ffffff;      /* 纯白文字 */
--text-secondary: #8a8a8a;    /* 灰色辅助文字 */
--code-bg: #1a1a1a;           /* 代码片段背景 */
```

### 字体系统

| 用途 | 字体 | 字号 | 字重 |
|------|------|------|------|
| **主标题** | JetBrains Mono | 72-96pt | Bold |
| **副标题** | 苹方-简 | 36-48pt | Medium |
| **正文** | 苹方-简 | 28-32pt | Regular |
| **数字强调** | JetBrains Mono | 120pt+ | Bold |
| **代码片段** | JetBrains Mono | 24pt | Regular |

### 排版系统（瑞士国际主义）

- **网格系统**: 12列网格，严格对齐
- **左对齐**: 拒绝居中，强化技术感
- **留白**: 大量负空间，信息密度 > 装饰
- **层级**: 通过字号/颜色/字重区分，不用下划线
- **对比**: 高对比度确保可读性

### 封面公式（新）

```
[深色背景 #0a0a0a]
    ↓
[霓虹青 #00f5ff 数据流/网格线装饰]
    ↓
[左侧：巨大数字 "3h/day" 动态模糊效果]
    ↓
[右侧：半透明Python代码片段]
    ↓
[底部小字："不是教程，是认知增强实验"]
```

### 5页内页结构

#### Page 1: 痛点页（物理代价）
- **不描述问题**，展示"时间贫困"的物理代价
- **核心公式**: `碎片时间 × 上下文切换成本 = 认知破产`
- **视觉**: 公式用霓虹青高亮，数据可视化

#### Page 2: 反常识页（核心洞察）
- **批判对象**: "教程思维"的低效性
- **核心洞察**: "买模板的人99%不会用，因为缺少触发器设计"
- **金句**: 这是你区别于99%博主的核心

#### Page 3: 解决方案页（Agent配置）
- **展示**: OpenClaw/其他Agent的实际配置截图
- **关键**: 打码敏感信息，但保留结构
- **强调**: "无人值守"系统 vs "半自动"工具

#### Page 4: 效果页（分屏对比）
- **不用文字**: 不用"节省2小时"这种空话
- **用对比**:
  - 左：手动操作录屏（10倍速，展示痛苦）
  - 右：Agent自动执行（实时，展示流畅）
- **视觉冲击**: 对比产生震撼

#### Page 5: CTA页（特权型钩子）
- **旧版（废弃）**: "评论区扣'1'领取"
- **新版（强制）**: 
  ```
  "这个配置涉及公司信息安全，我不能公开。
  如果你正在用AI重构工作流，
  私信'炼金术'，
  我发你脱敏版的触发器架构。"
  ```
- **效果**: 筛选高意图 + 规避算法 + 稀缺感

## 🗣️ 语言风格（极客身份）

### 禁用（旧版痕迹）

| 禁用表达 | 原因 |
|---------|------|
| "首先...其次...最后" | 学生作文体，太幼稚 |
| "建议大家收藏" | 太普通，无身份感 |
| "今天给大家分享" | 教程博主语气，太低级 |
| emoji轰炸 | 每页最多1个功能性emoji |
| 长段落 | 必须短句，技术文档风格 |

### 强制使用（技术隐喻）

| 旧表达 | 新表达 | 隐喻说明 |

| 脚本 | Agent注入 | 代码属性 |
| 设置 | 编译配置 | 程序员语言 |
| 循环 | 递归优化 | 算法思维 |
| 触发 | 事件劫持 | 黑客技术 |

## 📝 内容公式（新）

### 每篇笔记必须包含

1. **反常识洞察**
   - 挑战主流认知
   - 揭示隐藏真相
   - 建立权威感

2. **Agent配置**
   - 可执行的代码/配置
   - 截图证明真实性
   - 打码但保留结构

3. **幕后揭秘**
   - 展示自动化过程
   - 分屏对比效果
   - 制造技术神秘感

### 标题公式（新）

**悬念结构**:
```
"我用[方法]实现了[效果]，但[反转让用户好奇]"

示例：
"我用3个Python脚本每天偷回3小时，但第4个让我差点被开除"
"我用AI Agent重构了认知管道，但第一个版本让我损失10万"
"我编译了一套无人值守系统，但触发器设计差点触发公司警报"
```

**其他公式**:
- "不要再[旧方法]了，直接[新方法]"
- "99%的人[错误行为]，因为[真相]"
- "[惊人数字]的背后，是[反常识洞察]"

## 🔄 制作流程（5页手术版）

```
Step 1: 选题 → 找反常识洞察（而非工具清单）
Step 2: 调研 → 找真实Agent配置案例
Step 3: 大纲 → 5页手术版结构
Step 4: 文案 → 技术隐喻 + 短句
Step 5: 封面 → 赛博朋克视觉生成
Step 6: 检查 → image工具确认
Step 7: 发布 → 特权型CTA
Step 8: 复盘 → 筛选私信质量
```

### 与传统流程的差异

| 步骤 | 旧版（GLIMMER） | 新版（赛博朋克） |
|------|----------------|------------------|
| 选题 | 工具清单 | 反常识洞察 |
| 文案 | 7页，教程式 | 5页，技术隐喻 |
| 封面 | 荧光黄，大字报 | 深色，代码+数据 |
| CTA | 评论区扣"1" | 私信特权钩子 |
| 目标 | 收藏点赞 | 筛选高意图用户 |
