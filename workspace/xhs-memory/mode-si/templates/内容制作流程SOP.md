# 内容制作流程SOP（优化版）
## 从选题到发布的标准化流程

**版本**: v2.0（基于实际验证优化）  
**适用模式**: GLIMMER + 纳瓦尔认知成长模式  
**验证状态**: ✅ 已通过6篇笔记验证

---

## 流程总览

```
选题 → 调研 → 大纲 → 文案 → 封面 → 检查 → 发布 → 复盘
(30min) (45min) (20min) (90min) (15min) (5min) (10min) (持续)
```

**总耗时**: 约3.5-4小时/篇（熟练后可压缩至2-3小时）

---

## 阶段1: 选题（30分钟）

### Step 1.1: 从选题库选择
- 打开 `planning/TOPIC_LIBRARY.md`
- 查看当前选题状态（待发布/已发布）
- 根据以下标准筛选：
  - 与当前热点相关度
  - 个人经验/素材充足度
  - 竞品近期表现（避免同质化）
  - 制作难度评估

### Step 1.2: 确定模板类型
根据选题选择正文模板（8种之一）：
- **工具推荐** → 数字清单型
- **方法论** → 痛点解决型
- **观点输出** → 反常识观点型
- **人设故事** → 故事启发型
- **产品推广** → 产品转化型
- **热点借势** → 热点结合型
- **对比评测** → 对比反差型
- **话题讨论** → 提问互动型

**输出物**: 确定选题 + 选定模板类型

---

## 阶段2: 调研（45-60分钟）

### Step 2.1: 执行搜索
```bash
cd ~/.openclaw/skills/xhs-autopilot
python3 scripts/search/run.py \
  --keyword "选题关键词" \
  --output_dir ~/.openclaw/workspace/xhs-memory/mode-si/surveys/RESEARCH_ARCHIVE/$(date +%Y%m%d)
```

### Step 2.2: 资料收集
- 阅读相关书籍/文章（至少2-3篇）
- 搜索同类爆款笔记，分析结构
- 收集：
  - 金句（3-5条）
  - 数据（支撑观点）
  - 案例（1-2个真实案例）
  - 个人经历（增强可信度）

### Step 2.3: 观点提炼
- 总结3-5个核心观点
- 找到独特的切入角度
- 确定反常识点/金句爆破点

**输出物**: 资料包（金句+数据+案例+个人故事+调研截图）

---

## 阶段3: 大纲（20分钟）

### Step 3.1: 使用纳瓦尔7页结构规划

```
Page 1: 封面页（GLIMMER视觉锤）
  └─ 标题：使用爆款标题公式
  └─ 副标题：一句话价值承诺

Page 2: 开篇故事
  └─ 个人经历/用户痛点（2-3句钩子）

Page 3: 核心观点
  └─ 反常识观点/金句（加粗显示）

Page 4-6: 方法论（3-5个技巧）
  └─ 技巧1：名称 + 详细说明 + 案例
  └─ 技巧2：名称 + 详细说明 + 案例
  └─ 技巧3：名称 + 详细说明 + 案例

Page 7: 总结 + 行动引导
  └─ 核心心法（一句话金句）
  └─ 福利引导（评论区互动）
```

### Step 3.2: 确定每页要点
- 每页只讲一个核心点
- 控制字数：每页100-200字
- 预留Emoji和分隔线位置

**输出物**: 内容大纲（7页结构+每页要点）

---

## 阶段4: 文案（60-90分钟）

### Step 4.1: 标题撰写（15分钟）
- 写3-5个备选标题
- 使用爆款标题公式：
  - 公式1: 数字+价值
  - 公式2: 痛点+解决方案
  - 公式3: 反常识+好奇心
  - 公式4: 人群+结果
  - 公式5: 悬念+数字
- 选择点击率最高的标题

### Step 4.2: 正文撰写（45-60分钟）
- 按选定模板结构写作
- 每段控制在3-5行（手机阅读友好）
- 关键信息加粗/变色
- 适当使用Emoji（但不过度）
- 多用"你"拉近距离
- 多用短句，易于阅读

**写作技巧**:
- Page 2（开篇）：用故事/痛点/数据抓住注意力
- Page 3（核心）：金句单独成行，制造记忆点
- Page 4-6（方法）：每个方法配1个案例
- Page 7（结尾）：金句收尾 + 明确行动引导

### Step 4.3: 结尾设计（10分钟）
- 金句总结（增强记忆点）
- 互动引导（提升评论率）：
  - "你最喜欢哪个技巧？评论区告诉我👇"
  - "你有过类似经历吗？聊聊看👇"
- 转化引导（如需）：
  - "想要完整版，私信'认知'领取💬"
  - "建议收藏，反复阅读📌"

**输出物**: 完整文案（标题+正文+结尾）

## 阶段5: 封面（15分钟）

### Step 5.1: Canvas制作
```bash
mkdir -p ~/.openclaw/workspace/xhs-memory/mode-si/surveys/RESEARCH_ARCHIVE/$(date +%Y%m%d)_封面

cd ~/.openclaw/skills/xhs-autopilot
python3 scripts/cover/run.py \
  --type glimmer \
  --title "主标题（10字内）" \
  --subtitle "副标题（15字内）" \
  --output ~/.openclaw/workspace/xhs-memory/mode-si/surveys/RESEARCH_ARCHIVE/$(date +%Y%m%d)_封面/cover.png
```

### Step 5.2: 设计检查
- 背景：荧光黄 RGB(225, 255, 0)
- 标题：黑色粗体，10字内
- 副标题：灰色，15字内
- **页码**：**单图不标，多图才标**
- 品牌栏：底部黑色条 + "MODEL-SI 知识库"

**输出物**: cover.png（1080×1440px）

**输出物**: 确认无误的封面图


### Step 7.3: 正式发布
```bash

  --input ~/.openclaw/workspace/xhs-memory/mode-si/memory/01_episodic/draft_YYYYMMDD.md
```

### Step 7.4: 发布后操作
- 发布后30分钟内自评（引导讨论）
- 2小时内回复所有评论
- 置顶评论引导收藏/关注

**首评模板**:
```
"你觉得哪个观点最触动你？评论区聊聊👇"
"建议收藏，反复阅读📌"
```

**输出物**: 已发布笔记链接

## 阶段8: 复盘（24-48小时后）

### Step 8.1: 数据记录
记录以下指标：

| 指标 | 数值 | 健康标准 | 优化方向 |
|-----|------|---------|---------|
| 曝光量 | ___ | - | - |
| 点赞数 | ___ | - | - |
| 收藏数 | ___ | >曝光×10% | 提升干货密度 |
| 评论数 | ___ | >曝光×5% | 优化互动引导 |
| 涨粉数 | ___ | >曝光×2% | 强化人设/IP |
| 点击率 | ___ | >8% | 优化封面/标题 |

### Step 8.2: 分析优化
- 对比预期vs实际数据
- 分析成功/失败原因：
  - 标题是否吸引人？
  - 封面点击率如何？
  - 内容是否解决痛点？
  - 互动引导是否到位？
- 记录优化方向

### Step 8.3: 更新选题库
- 标记已发布选题
- 根据反馈补充新选题
- 删除表现差的选题方向

**输出物**: 复盘记录 + 更新后的选题库

## 效率提升技巧

### 批量制作（周日准备）
- 一次性确定下周3-5个选题
- 批量收集资料
- 批量制作封面模板

### 模板复用
- 建立文案片段库（金句/案例/数据）
- 复用成功的封面版式
- 复用有效的互动话术

### 碎片时间利用
- 通勤时间：构思选题
- 午休时间：收集资料
- 晚间时间：集中写作

## 常见问题

**Q: 每天要花多少时间？**
A: 新手期4小时/篇，熟练后2-3小时/篇

**Q: 没有灵感怎么办？**
A: 从选题库选择，或拆解同类爆款

**Q: 数据不好怎么办？**
A: 分析原因（封面/标题/内容/时间），小范围测试优化

**Q: 如何保持持续输出？**
A: 建立素材库 + 批量制作 + 固定流程 + 养成习惯
