# 指令路由

之后我可能会给你

1. 很多论文的网址。这时候你需要依次访问这些网址，确定论文的下载地址，然后依次下载所有的论文。进行论文收集的任务。

2. 或者是具体的论文名字。这时候你需要确定具体的论文下载方式，然后依次下载所有的论文。进行论文收集的任务。

3. 某个研究领域。这时候你需要搜索这个领域的相关论文，最好是最新最好的论文，前沿的sota，领域近五年的标志性成果的，以及最新的人们在这个领域的尝试等等，然后依次下载所有的论文。进行论文收集的任务。
    - 整理出包含这个领域研究工作的血缘关系图谱，也就是所有主要的引用和被引用关系，基于，借鉴研究成果的关系图谱，目的是让我看到这个领域的主要研究方向，和实效性人们作出研究分支，关注的问题重点。你可以按照第一性原理进行适度的发挥。

4. 我正在解决的问题。这时候你需要搜索和这个问题相关的论文，搜索思考前人对于解决这个问题有什么思路，找到所有相关的论文，尤其是最新的尝试。你需要对这个问题可能的解决方案进行分类，然后对每个分类进行论文收集。进行论文收集的任务。
    - 整理出包含这个领域研究工作的血缘关系图谱，也就是所有主要的引用和被引用关系，基于，借鉴研究成果的关系图谱，目的是让我看到这个领域的主要研究方向，和实效性人们作出研究分支，关注的问题重点。你可以按照第一性原理进行适度的发挥。
    - 整理出针对这个问题所有的解决方案的论文图景，目的是向我展示目前为止前人对这个问题所有的解决方式，（包含他们是如何设计实验，设计系统解决这个问题的），以及每种解决方式的优缺点，验证了关于解决这个问题的什么事实（包含由什么样的实验证明），找到了提出了什么可能的方向。
    - 你需要不断补全这两个图谱，不断进行搜集整理。
    - 上面两个图谱和对应的论文收集工作完成之后，进行下一步。
    - 然后你需要根据上面执行的结果，把这个问题的可能解决方向进行分类，展现出来整个系统的架构图，以及每个模块的详细内容，并且针对这个每一个问题，每一个问题对应的模块，展开跨领域的横向搜索，看看解决这个问题有没有什么跨领域的方法可以借鉴，这部分需要你弄清楚事情的本质，这个模块，或者这个问题的本质，看看在功能层面有没有什么别的领域的最新最好的成果，或者某些具体的步骤，有没有跨领域的更优代替。

5. 整理出来目前所有这个领域需要解决，还没有被解决的问题。写成一个文档。

# 论文收集

看一下这个论文收集文件夹的文件组织性质。你需要把每一个论文的pdf挨个下载下来，并且按照论文名字加上arxiv编码的形式命名。在所有论文下载命名完毕之后，你需要阅读每一篇论文，每一个论文写一个和pdf同名的md，其内容在 [论文整理的标准格式.md](论文整理的标准格式.md) 中定义，要求内容详细。确保内容来源可靠，有效。如果出现你找不到原文的状况，就直接告诉我，无论如何不要写出来不确定的内容。如果有不确定就写出来你不确定。确保你完整阅读整个论文的所有文字和图片，你也许需要使用某种工具来及逆行完整pdf的阅读。你需要阅读每一个论文完整的内容，里面的实验数据图像等等，你的最后输出的md的完整程度要达到极其详细。
